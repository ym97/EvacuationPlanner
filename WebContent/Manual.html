User Manual

Input

1. Static Data -

Infrastructure of the building in terms of nodes(points) and edges(arcs) where, rooms, halls are denoted by nodes and corridors, staircases are denoted by arcs. Nodes' static information contains -
node id(a unique number for each node), node name(description about the node), node capacity(number of people the room/hall can contain at a time), travel time, latitude and longitude.
The Arcs' information contains information about arc id, arc name, from node, to node, travel time, capacity.

2. Dynamic Data -

The dynamic data contains initial capacity of the nodes i.e. actual number of people present at the time at that node.
This data is to be fed when evacuation planning is implemented.

3. Scenario Data -

Implementing evacuation plan for different scenario like bomb blast or fire in a building, we need scenario information. It contains danger level(how much danger is there in a node because of disaster happened), waiting time(how long people can stay in a node safely)



Visualization

Visualization shows floor wise building which contains following options

1. Show Paths -

On click a source node, path going out from source node turns green.

2. Scenario in a timestamp -

You can see situation of a building on a particular timestamp, There is a option called show path with prev and next time option.Initially it sets to zero time that is evacuation process is not started yet. When click on next, it shows the path covered from time zero to one. Same applies for prev option.

3. Capacity View -

Increase node size according to the actual capacity of node (in scale down form).

4. Normal View -

Show equi-size node and edge graph of a building.

5. Reset Graph -

Reset Graph to zero time situation

6. Floor Number -

It shows total number of floor present in a building, Click on next to previous floor to observe from source to destination.
